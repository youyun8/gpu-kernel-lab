# 第一個 kernel

我們來寫第一個真正在 GPU 上跑的程式:vector add,$C = A + B$。它簡單到沒有任何優化空間,正好讓你把注意力放在「一段程式碼如何被幾千個 thread 同時執行」這件事上。

## thread / block / grid hierarchy

GPU 的並行度用三層結構描述。最小單位是 thread;threads 組成 block (同一 block 的 threads 能共用 shared memory、能 `__syncthreads ()`);blocks 組成 grid。你在 launch kernel 時指定 grid 與 block 的維度,硬體再把它們排到 SM/CU 上。

每個 thread 用內建變數算出「我負責哪一筆資料」:

$$
i = \text{blockIdx.x} \times \text{blockDim.x} + \text{threadIdx.x}
$$

## vector add

<PlatformTabs>
<Platform name="CUDA">

```cpp
// One thread per element. The grid is sized to cover n with a boundary guard.
__global__ void vectorAdd(const float* a, const float* b, float* c, int n) {
  int i = blockIdx.x * blockDim.x + threadIdx.x;
  if (i < n) c[i] = a[i] + b[i];
}

void launch(const float* a, const float* b, float* c, int n) {
  constexpr int kBlock = 256;
  int grid = (n + kBlock - 1) / kBlock; // ceil-div so every element is covered
  vectorAdd<<<grid, kBlock>>>(a, b, c, n);
  cudaDeviceSynchronize();
}
```

</Platform>
<Platform name="HIP / ROCm">

```cpp
// HIP mirrors the CUDA runtime API almost one-to-one.
#include <hip/hip_runtime.h>

__global__ void vectorAdd(const float* a, const float* b, float* c, int n) {
  int i = blockIdx.x * blockDim.x + threadIdx.x;
  if (i < n) c[i] = a[i] + b[i];
}

void launch(const float* a, const float* b, float* c, int n) {
  constexpr int kBlock = 256;
  int grid = (n + kBlock - 1) / kBlock;
  hipLaunchKernelGGL(vectorAdd, dim3(grid), dim3(kBlock), 0, 0, a, b, c, n);
  hipDeviceSynchronize();
}
```

</Platform>
</PlatformTabs>

<Callout variant="info" title="那個 if (i < n) 不是裝飾">
grid 用 ceil-div 算出來,最後一個 block 幾乎一定會有超出 n 的 thread。少了邊界檢查,這些 thread 會越界寫記憶體 — 在 GPU 上這常是 silent corruption 而不是乾脆的 crash。
</Callout>

## compile 流程

CUDA 用 `nvcc`,HIP 用 `hipcc`。兩者都會把 host 程式碼交給系統 C++ 編譯器,把 device 程式碼編成中介表示(PTX / GCN-CDNA ISA),最後在載入時 JIT 或 AOT 成實際 GPU 指令。

```bash
# CUDA
nvcc -O3 -arch=sm_80 vector_add.cu -o vector_add

# HIP (ROCm)
hipcc -O3 --offload-arch=gfx90a vector_add.cpp -o vector_add
```

本站的 kernels 用 CMake 包起來,自動偵測平台並帶入正確的 arch flag,你不必手記這些參數。

<LabBox path="kernels/01-basics">
編譯並執行 `vector_add`。它會配置兩個輸入 vector、在 GPU 上相加、和 CPU 的結果比對(correctness check),再用 benchmark harness 報告 achieved bandwidth (GB/s)與 % of peak。vector add 是純 memory-bound,你會看到 GFLOP/s 很低但 bandwidth 接近硬體上限 — 這是正常的,第 5 章的 roofline 會解釋為什麼。
</LabBox>
